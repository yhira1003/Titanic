# タイタニック生存予測

Kaggle タイタニックデータセットを用いた二値分類プロジェクト。  
特徴量エンジニアリングを施したうえで4種類の機械学習モデルを比較しました。

---

## 結果

| モデル | CV Accuracy | CV AUC | Train AUC | AUCギャップ |
|---|---|---|---|---|
| Logistic Regression | 0.8283 | 0.8763 | 0.8839 | 0.0076 |
| Decision Tree | 0.8148 | 0.8534 | 0.8761 | 0.0227 |
| Random Forest | 0.8384 | 0.8855 | 0.9722 | 0.0867 |
| **XGBoost** | **0.8384** | **0.8885** | **0.9444** | **0.0558** |

**最良モデル: XGBoost（CV AUC = 0.8885）**

---

## 主な発見

- **性別と称号（Title）が最も予測力の高い特徴量でした。**  
  女性乗客（Miss・Mrs）の生存率は70%超であるのに対し、  
  男性乗客（Mr）の生存率は20%以下でした。

- **家族人数は非線形な影響を持ちます。**  
  小家族（2〜4人）は一人旅や大家族（5人以上）より高い生存率を示しました。

- **Random Forestは過学習が最も大きいモデルでした（AUCギャップ = 0.0867）。**  
  XGBoostはほぼ同等のCV AUCを保ちながら、サブサンプリングによって  
  ギャップを0.0558まで抑制し、汎化性能で上回りました。

---

## 特徴量エンジニアリング

| 特徴量 | 内容 | 作成の理由 |
|---|---|---|
| `Title` | 名前から称号を抽出（Mr・Mrs・Miss・Rare） | 性別・年齢・社会的地位を1列に凝縮 |
| `Family_Size` | SibSp + Parch + 1 | 避難行動のグループ規模を表現 |
| `Family_Group` | Alone / Small / Large に分類 | Family_Sizeの非線形効果を捉える |
| `Fare_Log` | log(Fare + 1) | 右歪みの運賃分布を正規化 |
| `Fare_Band` | Fare_Logを4段階にビン分割 | 運賃の閾値効果を捉える |
| `Has_Cabin` | 客室記録があれば1、なければ0 | 客室記録の有無が生存率と相関 |

---

## 欠損値処理

| 列 | 欠損数 | 処理方法 | 理由 |
|---|---|---|---|
| Age | 177件（20%） | Pclass×Sexグループの中央値で補完 | グループ別に年齢分布が異なるため単純平均より現実に近い |
| Cabin | 687件（77%） | Has_Cabinフラグ列に変換・元列削除 | 欠損が多すぎて値の復元は困難。有無の情報自体を活用 |
| Embarked | 2件（0.2%） | 最頻値（S）で補完 | 件数が少ないため最頻値で十分 |

## モデルの詳細

### Logistic Regression
- ベースラインモデル。過学習がほぼなく安定（AUCギャップ = 0.0076）。
- 係数を確認すると Title_Miss・Title_Mrs が最も大きな正の値を持ち、  
  「女性称号であること」が生存に最も強く影響することを確認。

### Decision Tree
- 過学習の可視化を目的として使用。
- max_depth=3でCV AUCが最大となり、それ以降は深くするほど  
  Train AUCは上昇するがCV AUCは低下する、典型的な過学習パターンを確認。

### Random Forest
- バギングにより決定木単体より安定したスコアを達成。
- データ件数が891件と少ないため Train AUC が0.97まで上昇し、  
  AUCギャップが0.0867と最大になった。

### XGBoost
- 最良モデル。learning_rate（0.01・0.05・0.1・0.3）と  
  max_depth（3・4・5）のグリッドサーチでパラメータを選定。
- early_stopping（rounds=50）で最適な木の本数を自動決定。
- subsample=0.8・colsample_bytree=0.8で過学習を抑制。
